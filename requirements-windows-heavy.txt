# app -- including RVC/song -- runs in ONE conda env. `rvc/driver.py` defaults
# `worker_python=None` -> the RVC worker uses the same interpreter, not a second env.
# audio-separator declares numpy>=2 but that is METADATA ONLY: it imports + runs at

# do NOT bare-`pip install` audio-separator / g2p_en here -- that bumps numpy to 2.x
# and breaks GPT-SoVITS (the W4-b §6.3 smoke hit exactly this via g2p_en).
#
# Install ORDER matters (validated by W4-b gate 1 on the real machine, 2026-07-06):
#
#   1) swap the CPU onnxruntime for the GPU build (the two dists conflict):
#        python -m pip uninstall -y onnxruntime
#        python -m pip install -r requirements-windows-heavy.txt
#   2) torch + torchaudio come from the cu124 index, NOT default PyPI (a bare
#      `pip install torch` would fetch the wrong build) -- install SEPARATELY:
#        python -m pip install torch==2.5.1 torchaudio==2.5.1 --index-url https://download.pytorch.org/whl/cu124
#
# KNOWN pip-check inconsistency (accepted + recorded, W4-a §4.E / gate-1 §4):
# faster-whisper and rapidocr-onnxruntime declare the "onnxruntime" DIST name;
# after the GPU swap `pip check` reports it missing while runtime imports stay
# green (the module is provided by onnxruntime-gpu). Do NOT "fix" it by
# reinstalling the CPU package -- that would shadow the GPU EPs.
#
# tensorrt-cu12 install note: pypi.nvidia.com occasionally serves a transient
# sha256 mismatch on the -libs sub-wheel (seen in W4-a) -- RETRY once before
# treating a failure as real.
#
# numpy stays <2 in this env (GPT-SoVITS forward constraint); gate 1 verified the
# stack below does not bump numpy 1.26.4.
